01 Game Overview

Schmidley's Complex Summer is a fast-paced game where you play as a kid rushing to get to summer school before the bell rings. Each level has you sprinting through school halls and nearby streets, dodging obstacles like lockers, backpacks, and other students in your way.

The game features a clean white, brown, black, and yellow color scheme, giving it a simple but striking look. Quick reflexes and smart movement are key as you weave through tight spaces and avoid anything that might slow you down. Can you make it to class on time, or will you be late again?

02 Game Goal

The goal of the game is to reach the classroom before time runs out. Sprint through every level, collect books along the way, and don't let anything knock you off course.

✓ Win Condition
Reach the classroom before the timer hits zero.
✗ Lose Condition
Health reaches zero OR time runs out.

03 Player Controls

  • ← Left Arrow Move the player left
  • → Right Arrow Move the player right
  • ↑ Up Arrow Move the player forward

04 Game Mechanics & Logic

Main Actions:

  • Sprint through school halls and streets
  • Collect books for points
  • Dodge obstacles — lockers, backpacks, other students

Variables:

Variable Name Starting Value What It Does
score0Tracks how many points the player has earned
health3How many hits the player can take
timeRemaining60 secondsHow much time is left to reach the goal

If / Then Rules:

  • If the player collects a book → add 10 points
  • If the player hits an obstacle → lose 1 health
  • If health equals 0 → the game ends (lose)
  • If time equals 0 → the game ends (lose)
  • If the player reaches the classroom → the player wins

05 Game Loop

These steps repeat over and over while the game is running:

  1. Check for player input (did the player press a key?)
  2. Move the player based on the input
  3. Check for collisions (did the player touch anything?)
  4. Update score or health based on what was touched
  5. Check win or loss conditions
  6. Repeat until the game ends

06 Playtesting & Changes

Changes made after testing the game with real players:

⚠ Problem Found

Players said the game was too easy.

✓ Changes Made

Added more obstacles and reduced starting time from 80 to 60 seconds.

08 Designer Reflection

The hardest part was making the game balanced. Testing helped us change the timer and obstacles to make it feel fair and exciting without being frustrating.

Our game uses core programming logic — variables, if statements, and loops — to make everything work together. Building this showed us how game design and coding connect in real ways.
